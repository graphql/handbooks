Intended audience: new TSC members.

**Welcome to the GraphQL TSC! It's great to have you aboard!**

Over the coming days you will be granted various roles and access, in particular
you will be granted access to the `tsc-private` mailing list which is used for
announcing TSC votes and other important private messages, and the `TSC` role in
the GitHub organization.

## New member actions

Please take the following actions:

- Confirm you have access to the `tsc-private` mailing list (_everything that
  you find in this mailing list should be treated as confidential TSC
  information_; for access, email: operations@graphql.org)
- Confirm you have access to the private
  [TSC repository](https://github.com/graphql/tsc) (_everything that you find in
  this repository should be treated as confidential TSC information_; for
  access, contact an existing TSC member)
- Update your
  [contact details](https://github.com/graphql/tsc/blob/main/CONTACT.md) for
  other TSC members to contact you
- Review recent discussions

## Optional access

You can request access to:

- The `TSC` role on [the GraphQL Discord](https://discord.graphql.org), granting
  you access to the #tsc private channel (for access, ask in #moderation-etc)
- Become a Discord moderator (ask in #moderation-etc)
- Membership on the `security@graphql.org` mailing list (only request this if
  you need it and will help to action it, ask via operations@graphql.org)
- The `TSC` group on the Signal messaging app - typically only used for comms
  when we're in the same city, for example at GraphQLConf (ask Lee)
